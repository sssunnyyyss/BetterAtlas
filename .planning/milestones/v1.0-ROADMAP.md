# Milestone v1.0: Program Toggle Accuracy

**Status:** SHIPPED 2026-02-26
**Phases:** 1-1
**Total Plans:** 2

## Overview

This milestone delivered deterministic program variant behavior in catalog program mode, eliminating inaccurate major/minor toggles and stabilizing deep-link URL behavior.

## Phases

### Phase 1: Program and Major Toggle Accuracy

**Goal:** Program mode correctly maps and switches major/minor variants for the same program family without inaccurate fallback behavior.
**Depends on:** Nothing (first phase)
**Plans:** 2/2 plans complete

Plans:
- [x] 01-01: Define variant-selection contract and tighten API/frontend matching logic
- [x] 01-02: Wire toggle UI behavior to deterministic selectors and add regression tests

**Details:**
- Backend now uses strict-name-first family resolution with normalized fallback only when strict candidates cannot satisfy both kinds.
- Variant ordering is deterministic and degree-aware for stable frontend selection.
- Frontend uses shared selector utilities for options/toggles/tab canonicalization.
- Program mode applies bounded AI-summary relevance boosting with deterministic tie-breaking.
- Regression tests cover backend variant logic and frontend program-mode interactions.

## Milestone Summary

**Decimal Phases:**
- None

**Key Decisions:**
- Use strict-family matching before normalized fallback to reduce incorrect variant jumps.
- Preserve prior major/minor intent when toggling kinds and apply degree-aware deterministic fallback.
- Canonicalize invalid or missing `programTab` query values to `required`.
- Use AI summary/highlights as bounded ranking signals, not hard filters.

**Issues Resolved:**
- Major/minor toggles no longer depend on arbitrary first-result picks.
- Program dropdown no longer hides valid program families through BA-only filtering.
- Program-mode URL deep links are normalized to stable query state.

**Issues Deferred:**
- Explicit user-facing degree variant picker.
- Explanatory UI for missing counterpart variants.
- Wishlist/social scheduling roadmap work from initial project brief.

**Technical Debt Incurred:**
- Milestone was completed without a formal `v1.0` milestone audit artifact.
